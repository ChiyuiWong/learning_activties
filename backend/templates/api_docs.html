COMP5241 Group 10 - API Documentation

Note: All API endpoints must be prefixed with /api. For example, use /api/health instead of /health.

Available Endpoints

Health Check

GET /api/health - Check if the API is running

Security

POST /api/security/login - Login
POST /api/security/logout - Logout (Note: Use POST method, not GET)

Learning Activities

GET /api/learning/polls - Get all polls
POST /api/learning/polls - Create a poll
GET /api/learning/quizzes - Get all quizzes (not "quizs")
GET /api/learning/wordclouds - Get all wordclouds
GET /api/learning/shortanswers - Get all short answers
GET /api/learning/minigames - Get all minigames

Common Issues

Issue Solution
404 Not Found Make sure the endpoint includes the /api prefix.
405 Method Not Allowed Check if you're using the correct HTTP method (GET, POST, etc).
Failed fetch Make sure the backend server is running.
